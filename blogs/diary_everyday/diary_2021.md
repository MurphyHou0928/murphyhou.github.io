            - 因为中断程序的执行是根据中断标志位来调用的，设备以一定的时钟周期判断中断标志位的值，当中断标志位符合要求时，中断会执行；

            - 因为这while(1);的循环什么都没干，就是停留在while这句话；
            - 当这不是像C程序在运行时，只做这一件事情，要把他考虑成一个大的系统，虽然停在了这句话上，但是80C51还是在执行中断程序。

  - ```assembly
    
    ;DOS有很多中断子程序，其中21号中断的编号为9的功能就可以打印字符串到DOS。
    
    DATA SEGMENT
    STRING DB 'HELLO WORLD!', 13, 10, '$'
    ;13是回车    
    ;10是换行    
    ;'$'表示已经到了字符串尾，不在打印字符了。
    DATA ENDS
    
    CODE SEGMENT
    ASSUME CS:CODE,DS:DATA
    START:  
    MOV AX, DATA
    MOV DS, AX
    ;其实就是把DATA转到 DS 数据段寄存器，让数据段指向正确的位置。
    ;DATA不是指令，而是伪指令，实际上是一个动态的内存地址，要想运行，必须先放入到DS中，但是内存数不能放入段地址。
    ;MOV AX,DATA起到过渡作用
    
    MOV DX, OFFSET STRING ;获得字符串的偏移地址
    
    MOV AH, 09H
    INT 21H ;INT 21H是指令自动转入中断子程序的入口
    
    ;调用9号DOS功能（显示字符串 ）
    MOV AH, 4CH
    INT 21H
    ;调用程序结束功能
    CODE ENDS
    END START 
    ```

# 2021.9.13

## 对这个实验课老师有了新的认识

- 【记录】
  - 今天还是继续上次的电路设计实验，上次的进度是能使电路闪光但是没有呼吸效果，根据之前的仿真修改之后还是不行，于是另一个同学根据仿真的电路波形，提出了一种方法，就是加大与LED串联的电阻的阻值，因为这个电阻做限流作用，当电阻阻值增大后，这条线的电流会减小，电流不能超过LED的最大电流，这样的话LED就不是常量，呼吸效果也能体现出来了；
  - 我是率先完成了电路的修改，然后成功了；
  - 之后我又去帮着生哥检查电路，仿真电路和PCB设计电路与实物进行对照，发现有个电容引脚的焊点的焊球太大了，导致连到另一条线路上了，再做了修改之后，发现呼吸效果出来了，但是问题是呼吸周期太长了，且对电源的电压值有要求，只有一个特定的电池能触发呼吸效果，这个问题比较迷惑，一直没有解决；
  - 然后我又去给森哥检查电路，发现森哥的电路上很多线路都是断了的，且很多元件的引脚明明已经焊在了线路上，却仍然不导通；
    - 首先是元件引脚问题，我将元件引脚所焊接的铜线路剥离出来然后，重新焊接，使得引脚和线路通过锡球连接起来，这样就保证了引脚一定连在了线路上；
    - 线路中很多都是断了的，这个问题首先是，可以将断了的线路用锡补起来，但是我操作了一番太麻烦了，且效果不好，因为融化的锡很难画出一条直线，于是采用飞线的方式，修复了三条断线比较严重的线路；
    - 这样修改之后，发现电路常量，这样至少保证了电路基本上正常工作了，在检查一番后，发现电路板摔一下子，就能实现一次呼吸，仅有一次，那肯定怀疑是虚焊问题，我让森哥自己去检查一下哪里有问题，然后我去帮祥哥去看电路板了；
    - 但是很长时间森哥也没有找到问题所在，于是我想了一下，摔得一下子完成了充放电，很可能是电容得问题，然后重点检查了电容，发现一颗电容确实有点松动，在补焊了一下之后，电路正常工作，呼吸效果明显！
  - 祥哥电路板的问题也是我对照着PCB设计电路从而找出来的，是电容引脚焊接的时候焊到了旁边的电路上，找到问题，轻松秒杀；
  - 子硕的电路我检查的时候一直没有检出出什么问题来，焊接的也没有什么问题，但是一对照仿真电路查看就找到问题了，好几个电阻他焊错了，但是改了之后还是没有解决，很长时间都没有解决了这个问题，直到不经意间，将电路板倒了过来，板子就正常工作了，那毫无疑问是虚焊，补焊了一下后就解决问题了；
